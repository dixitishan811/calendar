fix: correct typo in EventResponseItem and ensure unique color generation

This commit addresses two issues:

1.  **Typo Correction:** Corrects a typo in the `EventResponseItem` data class, changing `calenderName` to `calendarName` for consistency.
2.  **Unique Color Generation:** Modifies the `convertStringToColor` function calls in both `calendarExtension.kt` and `eventExtension.kt`. Now, the color generation for calendars uses `id + name`, and for events, it uses `calendarId + calendarName`. This change aims to produce more distinct colors by incorporating more unique identifiers into the string used for color generation.
