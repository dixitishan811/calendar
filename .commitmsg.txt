feat: replace DummyCalendarApiService with RemoteCalendarApiService for calendar and event data

This commit transitions the application from using a dummy data source to a remote API for fetching calendar and event data. Key changes include:

-   **Removal of DummyCalendarApiService**: Deleted `DummyCalendarApiService.kt` and `CalendarApiService.kt`, which provided dummy calendar and event data.
-   **Introduction of RemoteCalendarApiService**: Added `RemoteCalendarApiService.kt`, which uses Ktor's `HttpClient` to interact with the Google Calendar API.
-   **Integration with ViewModel**: Updated `CalendarViewModel.kt` to utilize `RemoteCalendarApiService` instead of `DummyCalendarApiService` for fetching user calendars and events.
-  - Now fetch calendars using `fetchCalendarsForUser`
- - Now fetch events using `fetchEventsForCalendar`
-   **Error Handling**: Implemented basic error handling in `CalendarViewModel.kt` to log errors when API calls fail.
- **Data loading**: Added support for loading data in initialisation of Calendar Screen.
- **User ID**: Fixed user ID to be `user_id` instead of being random.
