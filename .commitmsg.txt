feat: refactor data layer for local and remote data sources, implement holiday API integration

This commit refactors the data layer to use separate packages for local and remote data sources and integrates the holiday API. Key changes include:

-   Moving local data entities to `data.localDataSource.model` and updating references.
-   Introducing remote data source components like `HolidayApiService`, `ClientWrapper`, `Result`, and `DataError`.
-   Implementing API integration to fetch holiday data from the remote source.
-   Creating new data models for remote holiday data: `HolidayResponse`, `Holiday`, `Date`, `Datetime`, `Timezone`, `Country`, `Meta`, `Response`.
-   Updating `CalendarViewModel` to load holidays from the API and store them locally.
-   Implementing `parseDateTime` function for parsing date and time strings.
-   Adding `buildkonfig` plugin for API key management and setting Json configuration in di.
-   Adding new network `Error` classes for better error handling.
- Update swipeable calendar to take holidays as not null.
